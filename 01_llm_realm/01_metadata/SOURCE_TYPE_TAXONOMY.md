# Source Type Taxonomy

Use this taxonomy when registering, mapping, or indexing source material.

| Source type | Definition | Typical metadata |
|---|---|---|
| interview | Interview recording, transcript, or notes | participant, date, location, interviewer, consent status |
| fieldnote | Researcher observation or field memo | author, date, site, context |
| academic_article | Journal article, book chapter, working paper | author, title, year, DOI or URL |
| policy_document | Policy text, regulation, guidance, consultation paper | institution, date, jurisdiction, version |
| institutional_report | Report from organization, NGO, company, agency, or public body | institution, date, title, URL or archive path |
| news_article | News, magazine, or wire item | outlet, author, date, URL, capture date |
| web_capture | Saved webpage, scrape, screenshot, or HTML capture | URL, capture date, publisher |
| legal_text | Law, case, contract, decision, legal brief | jurisdiction, authority, date, citation |
| dataset | CSV, spreadsheet, database export, structured data | creator, date, schema, license |
| image_or_scan | Image, handwritten note, scan, visual document | creator, date, OCR status, description |
| audio_video | Audio or video source | creator, date, duration, transcript status |
| correspondence | Email, letter, message, or exchange | sender, recipient, date, thread context |
| researcher_note | Researcher's private note outside protected writing drafts | author, date, project context |
| draft_or_writing | Researcher's active writing | author, date, protected status |
| external_query_result | Source fetched outside the Root Vault under configured policy | query, tool, date, URL, logging status |

## Rules
- Do not treat source type as evidence strength. Evidence strength is assigned separately.
- If a source belongs to multiple types, choose the primary type and add secondary tags.
- External sources must remain visibly marked as external unless moved into the Root Vault by the researcher.
